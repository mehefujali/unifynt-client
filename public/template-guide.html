Publisher's Manual

Comprehensive guide to creating, registering, and deploying professional website templates on the Unifynt Ecosystem.

Workflow Overview

Publishing a new template requires synchronization between the Frontend UI Registry and the Backend Database. Follow these specific engineering steps to ensure a seamless "Publish" experience for the end-user.

1

Frontend Component Architecture

Create a new module within the templates directory. Ensure all components use the standard prop interfaces for dynamic rendering.

// Path

src/components/templates/[template-id]/
  • Implement index.tsx as the primary entry point.
  • Maintain dynamic state using liveContent and liveTheme.
2

System Registration

To make the template selectable in the Admin Panel, you must register its metadata in the global registry.

Modify: src/components/templates/registry.ts

export const TEMPLATE_REGISTRY = {
  // Add your new template here
  "your-template-id": {
    id: "your-template-id",
    name: "Business Elite",
    description: "Professional description...",
    thumbnail: "/templates/thumb.jpg",
    component: YourTemplateComponent
  }
};
3

Database Synchronization

Mandatory

The backend uses Foreign Key Enforcement. A template ID selection will fail to save if it does not exist in the database's SiteTemplate table.

Registration Script

Create a temporary script register.ts in the server root:

await prisma.siteTemplate.create({
  data: {
    templateCode: "your-template-id",
    name: "Display Name",
    isActive: true
  }
})

Critical Sync: The ID used in the Frontend Registry MUST match the TemplateCode in the Backend Database.

Go-Live Checklist

Build Verification

Ensure no TypeScript or ESLint errors in the new template folder.

JSON Structure

Verify default-site-data.ts contains all new UI data points.

Publish Test

Select template in Admin and click "Publish" to verify DB persistence.
